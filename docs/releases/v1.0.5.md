# MX8 v1.0.5

## Highlights

- Refactored `mx8-py` internals by splitting the previous large `lib.rs` surface into focused source files (checkpoint, autotune, stats, loaders, and Python API entrypoints).
- Added an operator-facing design partner onboarding checklist to make launch readiness explicit and repeatable.
- Bumped crate/package versions to `1.0.5` for a consistent release surface.

## Reliability and Developer Experience

- Preserved existing behavior while improving code organization for safer follow-on changes.
- Kept release gates as the acceptance bar: format, lint, workspace tests, and smoke.
- Hardened lint compatibility in runtime/demo targets (`unexpected_cfgs`, targeted clippy allow) to keep gates green.
- Hardened Python/distributed gate scripts to avoid HA/state replay interference in single-coordinator restart tests.

## Docs and Onboarding

- Added `docs/design_partner_onboarding_checklist.md` and linked it from `README.md` and `docs/user_guide.md`.
- Kept deployment, runbook, and API docs as the primary onboarding references.

## Publish Status

- Version prepared: `mx8==1.0.5`
- Publish/tag/push execution is pending release-owner action.
